# Execution Plans

Storage for execution plans generated by `/discuss`. Path: `cursor-agent-team/ai_workspace/plans/`

---

## Quick Reference

| What | Path | Format |
|------|------|--------|
| Plan File | `plans/` | `PLAN-[TopicID]-[Seq].md` |
| Index | `plans/` | `INDEX.md` |

Examples:
- `PLAN-C-001.md` - Topic C, Plan 1
- `PLAN-A-002.md` - Topic A, Plan 2

---

## NEVER DO

- NEVER delete `INDEX.md`
- NEVER delete this README
- NEVER modify plan files without updating status
- NEVER create plans outside this directory

---

## Plan File Structure

```markdown
# Execution Plan PLAN-[TopicID]-[Seq]

## Plan Info
- **Plan ID**: PLAN-[TopicID]-[Seq]
- **Topic**: [TopicID] - [Topic Name]
- **Created**: YYYY-MM-DD HH:MM:SS
- **Creator**: /discuss
- **Status**: Pending / In Progress / Completed / Cancelled

## Goal
[Core objective]

## Steps
1. [Step 1]
2. [Step 2]
3. [Step 3]

## Related Files
- [File path 1]
- [File path 2]

## Expected Results
[Expected outcome]

## Notes
[Important notes]

## Execution Records
- [Time] - [Executor] - [Result]
```

---

## Status Values

| Status | Meaning |
|--------|---------|
| Pending | Not yet started |
| In Progress | Currently executing |
| Completed | Successfully finished |
| Cancelled | Cancelled |

---

## Workflow

1. `/discuss` generates plan → saves to `PLAN-[TopicID]-[Seq].md`
2. `/discuss` updates `INDEX.md`
3. `/crew PLAN-XXX-XXX` executes plan
4. Executor updates plan status and execution records
