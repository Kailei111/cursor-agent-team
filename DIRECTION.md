# Direction

> These are directions, not commitments. The project evolves based on real needs, not predetermined timelines.

## Current Status (v0.10.11)

cursor-agent-team has reached **reference architecture** level for single-conversation, multi-role human-AI collaboration:

- Core methodology implemented and validated
- Academic preprint completed (ready for arXiv)
- Product positioning clarified

## Potential Directions

The following are areas we may explore. No timeline, no guarantees.

### Protocol & Specification

- **Agent Workspace Spec**: Formalize the plans/notes/cards directory structure as a reusable specification
- **Phase Markers Protocol**: Abstract phase markers as a state machine protocol, not just prompt convention
- **Spec-Script Interface**: Define tool signatures and return formats for cross-platform portability

### Documentation & Patterns

- **Pattern Blog Series**: Each pattern (Minimal Handoff, Persona Sandboxing, Aspect-Oriented Prompting, etc.) as a standalone article
- **Implementation Guide**: Step-by-step guide for building similar systems on other platforms

### Platform Exploration (Low Priority)

- **VS Code / JetBrains**: Investigate feasibility of porting core concepts
- **Web-based Agent Platform**: Explore integration possibilities

## Not Planning

We are explicit about what we're **not** pursuing:

- Multi-agent parallel orchestration (different paradigm)
- Cloud-scale deployment / SaaS
- Enterprise features (RBAC, audit logs, etc.)
- Commercialization

## Philosophy

> "First capital, then ideas" — we focus on methodology depth over feature breadth.

This project prioritizes:
1. Implementation completeness over wide compatibility
2. Clarity of thought over speed of iteration
3. Reference architecture over product deployment
